# General Mode Workflow

Run any task through MassGen's multi-agent system. Agents independently produce
solutions and converge through checklist-gated voting. Use this for tasks that
don't fit neatly into evaluate, plan, or spec — writing, code generation,
research, analysis, design, or anything where multiple perspectives improve
the result.

## Overview

General mode is the default. Use it when:

- The task doesn't clearly call for evaluation, planning, or specification
- You want multiple AI agents to independently tackle a problem
- The output is a deliverable (code, document, design) rather than a critique or plan

No fixed output schema — the output is whatever the agents produce for the task.

## Context File Template

Write `$WORK_DIR/context.md` with this structure. Only "Task" is required —
the other sections improve results but are optional.

```markdown
## Task
<what needs to be accomplished — clear, specific, actionable>

## Context
<relevant background: codebase overview, existing files, constraints,
target audience, tech stack, or any information that helps agents
produce better work>

## Quality Expectations
<what "good" looks like for this task — visual polish, completeness,
performance targets, tone, or other quality dimensions>
```

## Criteria

**Auto-generated by default.** When no `--eval-criteria` flag is provided,
MassGen generates evaluation criteria from the task content. This works well
for most general tasks.

**Custom criteria override**: If you have specific quality axes to enforce,
write criteria JSON to `$WORK_DIR/criteria.json` and pass via `--eval-criteria`.
See `references/criteria_guide.md` for the format.

## Output

The winner's answer is written to `$WORK_DIR/result.md`. Any files the agents
created are in the winner's workspace (the workspace path is shown in
`result.md` — look for "Workspace cwd" or check `status.json` in the log
directory for `workspace_paths`).

There is no fixed output schema — the output depends entirely on the task.

## Using the Output

1. **Read `result.md`** for the winning agent's summary of what was produced
2. **Copy deliverable files** from the winner's workspace to your project
3. **If the output contains a structured task list or action items**, consider
   grounding it in your native task system (same as plan/spec modes) — but
   this is optional for general mode since many tasks produce artifacts rather
   than task lists
